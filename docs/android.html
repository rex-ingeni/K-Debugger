Android 연결 가이드

K-Debugger(macOS)에 Android 기기·앱을 붙이는 방법입니다. 연결은 adb로 이뤄집니다.

1. 사전 준비

  1. adb 설치 — Android SDK Platform-Tools. 예: brew install android-platform-tools
  2. 기기에서 USB 디버깅 켜기 — 설정 → 개발자 옵션 → USB 디버깅 ON (개발자 옵션은 "빌드 번호" 7회 탭)
  3. USB로 연결 후 기기에 뜨는 "USB 디버깅 허용" 승인
  4. 확인: 
    adb devices
# List of devices attached
# XXXXXXXX   device   ← 이렇게 뜨면 정상
기기가 unauthorized로 뜨면 기기 화면의 허용 팝업을 승인하세요. 아무것도 안 뜨면 케이블(데이터 지원)·포트를 바꿔보세요.

2. 기기 감지

K-Debugger를 실행하면 adb로 연결된 기기를 자동 감지해 상단 기기 선택에 표시합니다. 여러 대면 거기서 대상 기기를 고릅니다. 연결은 앱이 유지하므로 화면을 새로고침해도 재연결이 필요 없습니다.

3. WebView 디버깅 지금 동작

앱 안의 WebView를 Chrome DevTools Protocol(CDP)로 직접 디버깅합니다. 별도 SDK 없이 adb만으로 동작합니다.

  1. 디버깅할 Android 앱의 디버그 빌드에서 WebView 디버깅을 켭니다: 
    if (BuildConfig.DEBUG) {
    WebView.setWebContentsDebuggingEnabled(true)
}
  2. 그 WebView가 있는 화면을 기기에서 연 상태로 둡니다.
  3. K-Debugger의 WebView 탭에서 탐지된 타깃(페이지)을 고르면 DevTools 프론트엔드가 임베드되어 열립니다.
WebView 타깃이 안 보이면: 디버그 빌드인지, setWebContentsDebuggingEnabled(true)가 호출됐는지, 해당 WebView 화면이 실제로 떠 있는지 확인하세요.

4. 로그 · 네트워크 · DB · Prefs (SDK 연동) 동작

앱의 로그·네트워크·DB·Preferences를 보려면 device측 KDebugger SDK를 연동합니다. SDK는 데스크톱과의 연결 · 이벤트 전송(sendEvent) · 명령 처리(commandHandler)만 제공하고, 각 플러그인은 앱의 후킹 지점에서 이 API로 데이터를 흘려보냅니다(Flipper의 후킹 지점을 미러링하는 방식).

연결

// Application.onCreate() — 디버그 빌드에서만
if (BuildConfig.DEBUG) KDebugger.init(this)   // abstract socket "k-debugger" 대기
# 데스크톱에서 소켓 매핑
adb forward tcp:8088 localabstract:k-debugger

프로토콜: 4바이트 빅엔디안 길이 헤더 + UTF-8 JSON. 릴리스에서 init을 안 부르면 no-op(오버헤드 0).

로그

로깅 파이프라인에 후크(로그 Tree/래퍼)를 심어 각 로그를 이벤트로 전송:

KDebugger.sendEvent("log", id, ts,
    JSONObject().put("level", level).put("tag", tag).put("message", msg))

네트워크 — REST

OkHttp 빌더에 인터셉터를 붙여 요청·응답을 가로채 전송(Flipper 네트워크 후킹과 같은 지점):

// OkHttpClient.Builder.addInterceptor(RestInterceptor())
// 인터셉터에서 요청/응답 메타·바디를 sendEvent("network", id, ts, {...}) 로 전송

네트워크 — WebSocket

WebSocket 리스너를 래핑해 프레임 단위로 전송:

// onOpen / onClose / onSend / onReceive 에서 각 프레임을 sendEvent("network", ...) 로 전송

DB

데스크톱이 보낸 SQL을 commandHandler에서 앱 DB에 실행하고 결과를 회신(요청↔응답 짝):

KDebugger.commandHandler = { cmd ->
    // cmd 의 SQL 을 앱 DB 에 실행 → JSONObject(rows/columns) 반환
}

Prefs / 상태

SharedPreferences 값을 조회해 sendEvent("prefs", ...) 로 스트리밍하거나 commandHandler 로 응답.

모든 후킹은 디버그 빌드에서만(if (BuildConfig.DEBUG)) 연결하세요 — 릴리스 빌드는 no-op(오버헤드 0)입니다.

참고

  • 서명 전 단계라 K-Debugger 첫 실행은 앱 우클릭 → 열기.
  • iOS는 아직 지원하지 않습니다(Android 전용).
